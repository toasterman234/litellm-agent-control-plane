---
title: "Claude Managed Agents"
description: "Run agents using Anthropic's Claude Managed Agents runtime."
---

Claude Managed Agents is Anthropic's hosted agent runtime — the platform proxies agent creation, session management, and event streaming through your LiteLLM gateway so your team never touches Anthropic credentials directly.

## Prerequisites

- LiteLLM Agent Platform running (`docker compose up` or deployed)
- Anthropic API key

## 1. Add your Anthropic API key

Open **Settings → Credentials** in the UI and add your Anthropic API key. The platform stores it encrypted and injects it at the wire level.

## 2. Register the runtime harness

If `claude-managed-agents` is not already listed under **Runtimes**, add it via the API:

```bash
curl -X POST $LAP_URL/api/runtime-harnesses \
  -H "Authorization: Bearer $MASTER_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "alias": "claude-managed-agents",
    "api_spec": "claude_managed_agents",
    "api_base": "https://api.anthropic.com"
  }'
```

## 3. Create an agent

In the UI, click **New Agent**, choose `claude-managed-agents` as the runtime, select a model, and save.

Or via the API:

```bash
curl -X POST $LAP_URL/api/v1/managed_agents/agents \
  -H "Authorization: Bearer $MASTER_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-claude-agent",
    "runtime": "claude-managed-agents",
    "model": "claude-opus-4-5",
    "system_prompt": "You are a helpful assistant."
  }'
```

## 4. Start a session

```bash
curl -X POST $LAP_URL/api/v1/sessions \
  -H "Authorization: Bearer $MASTER_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "<agent-id>"
  }'
```

## 5. Send a message and stream events

```bash
curl -X POST $LAP_URL/api/v1/sessions/<session-id>/events \
  -H "Authorization: Bearer $MASTER_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "human_turn",
    "content": "Summarize the latest changes in this repo."
  }'
```

Stream the response:

```bash
curl -N "$LAP_URL/api/v1/sessions/<session-id>/events/stream" \
  -H "Authorization: Bearer $MASTER_KEY"
```

Events follow the Anthropic Managed Agents SSE shape:

```
data: {"type":"session.status_running"}
data: {"type":"agent.message","content":"Here are the latest changes..."}
data: {"type":"session.status_idle"}
```

## SDK usage (Rust)

```rust
use lap_sdk::{Lap, LapConfig};

let client = Lap::new(LapConfig::anthropic(std::env::var("ANTHROPIC_API_KEY")?));

let agent = client.beta().agents().create(CreateAgentParams {
    name: "my-claude-agent".into(),
    model: "claude-opus-4-5".into(),
    ..Default::default()
}).await?;

let session = client.beta().sessions().create(CreateSessionParams {
    agent_id: agent.id.clone(),
    ..Default::default()
}).await?;

client.beta().sessions().events().send(&session.id, SendEventsParams {
    content: "What is the capital of France?".into(),
    ..Default::default()
}).await?;

let mut stream = client.beta().sessions().events().stream(&session.id).await?;
while let Some(event) = stream.next().await {
    println!("{:?}", event?.payload());
}
```

## Event kinds

| Event kind | Meaning |
|---|---|
| `session.status_running` | Agent is processing |
| `agent.message` | Text output from the agent |
| `agent.tool_use` | Agent is calling a tool |
| `agent.tool_result` | Tool result returned |
| `session.status_idle` | Turn complete |
| `session.error` | Runtime error |
